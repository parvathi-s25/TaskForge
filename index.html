Why Priority Queue Wins

  • Starvation prevention: the aging system automatically promotes long-waiting low-priority tasks, so nothing sits in the queue forever.
  • O(log n) insert/extract: the binary min-heap keeps insertion and extraction logarithmic, vastly outperforming a linear scan for the next task as the queue grows.
  • Mirrors real OS schedulers: production operating systems use priority-based heaps (with aging) for process scheduling for the exact same reasons.
